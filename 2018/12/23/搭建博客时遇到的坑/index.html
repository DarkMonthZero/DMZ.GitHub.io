  • 问题2

    每次部署以后都需要重新设置GitHub里面的自定义域名

解决过程：去网上查找原因，没有找到该问题的解决办法。大部分都说要把CNAME文件放到source文件夹中，结果我发现我的CNAME文件带的txt后缀，把后缀去掉以后还是不行。然后我发现当我使用hexo g命令以后生成的public文件夹里面CNAME内容为又去搜索，还是找不到解决办法，我怀疑是因为执行npm install hexo-deployer-git –save这个命令时!这个警告的问题，所以就开了VPN重新安装了，安装之后再运行任hexo指令就提示根据提示去对应文件夹里面找了一下，发现有个一模一样的文件夹，但是还是提示找不到，最后我把原来的文件夹重命名为提示的文件夹就不再出上边的提示了，并且hexo g命令之后public文件夹里面的CNAME文件正常了，再次部署以后Custom domin选项没有被清空。到此这个问题算解决了。

总结：这个问题困扰了我好几天，因为在网上搜不到解决方案，所以整个过程全都是“我寻思”解决的。只有参考意义，如有错误请指出。

  • 问题3

    访问网站时网页出错

    解决过程：清除浏览器缓存就可以正常访问了
