Oracle数据库

发表于 2019-08-17 | 分类于 数据库

Oracle数据库是甲骨文公司的一款关系数据库管理系统，本篇博客记录Oracle数据库的相关知识

Oracle数据库

Oracle数据库在sql plus中一些命令可以不用分号结尾，sql语句则一定要用分号结尾,Oracle数据库中也可以通过AS设置别名

解释数据库、表空间、数据文件、表、数据的最好办法就是想象一个装满东西的柜子。数据库其实就是柜子，柜中的抽屉是表空间，抽屉中的文件夹是数据文件，文件夹中的纸是表，写在纸上的信息就是数据。

表空间

表空间是数据库的逻辑存储空间，可以理解为在数据库中开辟了一个空间，用于存放数据库的对象。一个数据库可以有多个表空间构成。表空间是oracle区别于其他关系型数据库的重要概念，且Oracle数据库的优化大部分都是通过表空间实现的。表空间是由一个或多个数据文件构成的，数据文件的位置与大小由用户自定义。用户创建的表或数据库及其他对象都是存放在表空间中的数据文件中。

表空间的分类

在Oracle数据库中，表空间分为永久性表空间（永久表空间）、临时表空间、UNDO表空间。默认情况下，账号所拥有的表空间有6个，分别是：

SYSTEM，用来存放用户的表、试图等数据库对象，它是一个永久表空间。与USERS不同的是SYSTEM常用来存储系统信息

SYSAUX,用于EXAMPLE表空间的辅助表空间

EXAMPLE,用于安装数据库事例的表空间

UNDOTBS1,用来存储撤销信息，属于UNDO表空间

TEMP,用于存储SQL语句处理的表以及索引信息的，属于临时表空间

USERS,存储用户创建的数据库对象，属于永久表空间

永久表空间

永久表空间用来存放数据库中需要持久化的对象。例如数据表、试图、存储过程等

临时表空间

临时表空间则用来存放数据库操作中中间执行的过程，当执行结束后，存放的内容就会被自动的释放掉，不进行永久的保存

UNDO表空间

UNDO表空间是用来保存事务所修改数据的旧值，用来就行数据的回滚。

Oracle中的数据表

数据表表是存储数据时的一个基本存储单位，Oracle数据库中的表都是二维结构，由行和列组成，横向就是行，纵向就是列。行就是一条记录，列就是字段或者域。每张表的每一列数据都必须具有相同的数据类型，每一列的名字以及每一行记录都要是唯一的

数据表中的数据类型

Oracle中的数据类型分为字符型、数值型、日期型、其他类型。

  1. 字符型：

    CHAR(n)、 NCHAR(n)：定长类型，其中NCHAR是以Unicode存储的，且CHAR最大长度为2000，NCHAR最大长度为1000。通常情况下使用NCHAR存储汉字

    VARCHAR2(N)、NVARCHAR2(N)：变长类型，其中NVARCHAR@是使用unicode存储的，前者最大长度为4000，后者为2000

  1. 数值型：

    NUMBER(p,s)：其中p代表的是有效数字，s代表小数点后的位数，如果s为正数，代表小数点上最低有效数字的位数，如果为负数代表小数点上最大有效数字到小数点的位数。通常使用这个类型

    FLOAT(n)：主要存储二进制数据，能表达的二进制位数为1-126位，如果需要把这个二进制转换为十进制，就需要乘以0.30103。

  2. 日期型：

    DATE: 范围为4712年1月1日到9999年12月31日，DATE类型可以精确到秒，较常用

    TIMESTAMP:可以精确到小树苗

  3. 其他类型：主要用来存放大对象

    BLOB:可以存放4G的数据，以二进制形式存放

CLOB:容量为4GB，以字符串形式存放

数据类型是控制数据每一列的数据类型，而约束就是控制每一列是否满足某一条件，约束的作用是为每一列定义规则，确保数据的完整性

非空约束

非空约束：NOT NULL，非空约束可以在创建表、修改表时设置，在创建表时或修改表时在数据类型之后添加非空约束，默认情况下所有数据都是可以为空的。在修改字段为非空时，必须保证所有数据是没有控制的，负责会有异常。去除非空约束时，把NOT NULL 改为 NULL就可以了

主键约束

主键约束：PRIMARY KEY,确保表当中每一行数据的唯一性，设置主键约束的字段要求该字段是非空、唯一的，一张表只能设计一个主键约束，主键约束可以由多个字段构成（联合主键或复合主键），在数据类型之后加上主键约束就可以为该列添加主键约束

查询中的运算符与表达式

Oracle中的运算符与表达式：表达式是操作数与运算符来组合构成的，Oracle中的操作数可以有变量、常量及表中的字段组成。Oracle数据库中提供了算数运算符、比较运算符及逻辑运算符：AND(与),OR(或),NOT(非)，比较运算符的优先级大于逻辑运算符，而逻辑运算符的执行优先级为 NOT、AND、OR

在select语句中使用算数运算符不影响表中的数据，在Oracle数据库中，!=可以用<>代替，<>是标准语法，可以移植到其他任何平台，!=是非标准语法，可移植性差，但都是表示不等于的意思，应用中尽量用<>
