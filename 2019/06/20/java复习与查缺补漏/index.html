java复习与查缺补漏

遇到的问题

  • Scanner类在循环中必须每次重新创建它的实例才能重新获取键盘输入

  • for循环中的声明语句会执行一次，而条件与动作会执行多次

  • 遇到编译时错误提示参数异常，看API没找到参数哪里异常了，最后发现是倒错包了。

  • 从一个项目复制文件到另一个项目可能会导致文件乱码，这是因为两个项目的编码不一样。

  • 当一个类创建的时候，会递归调用父类的构造方法。

  • 当创建一个对象序列化输出流且不使用时，会有java.io.EOFException异常。

异常处理

  • 使用try..catch..finally来捕获异常，使用throw可以在方法体里抛出异常，在声明方法时通过添加throws关键词可以在调用这个方法时必须try..catch或者继续往上抛。可以通过继承Execption类来创建自定义异常类。可以通过printStackTrace来打印异常信息。当运行到抛出异常的代码时，后边的代码就不会再执行。

字符串

  • Java中的字符串具有不变形，每次试图改变字符串时都是创建了一个新的字符串对象。如果要频繁修改字符串，应使用StringBuilder类。

  • String类的常用方法：

    1. int length() 返回当前字符串的长度。

    2. int indexOf(char/String ch/str) 返回ch/str在字符串中第一次出现的位置。

    3. int lastIndexOf(char/String ch/str) 返回ch/str在字符串中最后一次出现的位置。

    4. String substring(int beginIndex,int endIndex) 获取从begin开始到end结束的字符串

    5. String substring(int beginIndex) 获取从begin开始到结尾的字符串

    6. char charAt(int index) 获取字符串中指定位置的字符

    7. String[] split(String regex,int limit) 分割字符串

    8. byte[] getBytes() 将字符串转换为byte数组

时间和日期的处理

  • 使用Calendar类进行时间和日期的处理：

    1. 通过getlnstance()来获取Calendar的实例

    2. 通过 get(Calendar.YEAR/MONTH/MINUTE..)来获取年月日等，创建一个int来接受返回值。

  • Calendar常用方法

    1. Date getTime() 获取Calendar的Date对象。

    2. Long getTimeInMillis() 获取此Calendar的时间值，以毫秒为单位。

Math类的常用方法

  1. long round() 返回四舍五入后的整数

  2. double floor() 返回小于参数的整数

  3. double ceil() 返回大于参数的整数

  4. double random() 返回[0,1]之间的随机数

集合

  • 泛型集合可以添加泛型的子类对象。

  • 集合可以使用contins(Object o)方法判断当前集合是否有对象o

  • 判断set集合时需要重写equals和hashCode方法

  • 集合中的sort方法判断字符串时会根据首字母依次往后推来判断，从大到小为：0-9,A-Z，a-z

  • Collection.sort()方法中的形参如果是引用类型，则必须实现Comparable借口

二进制

  • 位权： 一个数在不同的位置上所代表的值不同

  • 进制间的转换：

    1. 二进制转十进制：采用科学计数法，按权展开。以111001为例：
      111001 ->
      2^(6-1)+2^(5-1)+2^(4-1)+2^(1-1) ->
      32 + 16 + 8 + 1 = 57

    2. 十进制转二进制：采用短除2直到把商为0为止，把每次的余数倒过来看就是这个十进制数的二进制表现，按照这个方法短除8就只这个数的八进制表现.例如:57除2,商28余1。28除2,商14余0。14除2，商7余0,7除2，商3余1,3除2，商1余1。1除2，商0余1。倒过来就是111001

  • Java中八进制开头用0表示，十六进制开头用0x表示

  • 二进制的位运算：

    1. 按位与：

      • 按位与：两位全为1，结果才为1。例如： 51 & 5 先把十进制数转换为二进制，即0011 0011 & 0000 0101，结果为 0000 0001。因此 51 & 5 = 1。

      • 按位与可以用来清零一个单元，只要让这个单元与一个各位数都为0的二进制数相与，结果就为0

      • 按位与还可以取一个二进制数的指定位，例如x = 10101110，取X得低四位用X & 0000 1111 = 0000 1110。就可得到

    2. 按位或：

      + 只要有一个为1，结果就为1。例如： 51 | 5 -> 0011 0011 | 0000 0101, 结果为 0011 0111 ，因此 51 | 5 = 55。
      • 按位或可以对一些数据的某些位置1，例如：想X = 1010 0000，如果要把低四位的位置1，只需要把X | 0000 1111即可。得到结果 1010 1111。

    3. 异或运算：

      • 两个位置不同才为1，否则为0。例如 51 ^ 5 -> 0011 0011 ^ 0000 0101 = 0011 0110。 把结果转为十进制就是 54。

      • 异或运算可以使特定位翻转，例如X = 1010 1110，如果要使X得低四位翻转，只需让X ^ 0000 1111 即可得到 1010 0001。

      • 异或运算还可以与0相异或，来保留原值。异或还可以用来让两个变量交换值，如 A = 0011 B = 1100 经过A = A ^ B = 1111， B = A ^ B = 0011， A = A ^ B = 1100。两个数的值就交换了。

    4. 取反运算：

      • 对一个二进制数按位取反，0变为1 1变为0.例如： ！51 = ！0011 0011 = 1100 1100

    5. 左移运算:

      • 将一个运算对象的二进制位全部左移若干位,左边的二进制位丢弃,右边补0. 例如: 2 << 1 = 0010 << 1 = 0100 = 4

      • 若左移时舍弃的高位不包含1，则没左移一位，相当于该数乘2

    6. 右移运算：

      • 将一个数的格二进制位全部右移X位，正数左补0，负数左补1，右边丢弃X位。操作数没右移一位，相当于该数除以2。例如： 4 >> 2 = 0100 >> 2 = 0001 = 1。 -14 >> 2 = 1111 0010 >> 2 = 1111 1100 = -4

    7. 无符号右移：

      • 各个位向右移X位，右移后左边补X位0，丢弃右边X位。

    8. 二进制负数的表示

      • 负数以正数的补码形式表示，而说到补码就不得不提原码、反码

      • 原码是一个整数按照绝对值大小转换成的二进制数称为原码。例如：14的原码为00000000 00000000 00000000 00001110

      • 反码就是将原码按位取反所得到的就称为原码的反码。例如：14的反码为11111111 11111111 11111111 11110001。这时候原码和反码就互为反码。

      • 补码：反码加1称为补码。例如14的补码就是11111111 11111111 11111111 11110001 + 1 = 11111111 11111111 11111111 11110010

      • 例如：-14 << 2 = 1111 0010（前24为1） << 2 = 1100 1000然后1100 1000 - 1 = 1100 0111得到反码，反码取反得到原码0011 1000 原码对应的十进制数为： 56取相反数就得到-56。所以-14左移两位结果为-56

  • Java内置的进制转换

    • Java中的进制转换方法都是在integer这个类里边，十进制转其他进制使用toXXXX方法。其他进制转十进制使用parseInt方法。

  • java中的进制

    1. 在开发中，二进制主要用于跨平台，如：文件读写，数据通信。

    2. Java中的数据类型：1个字节是8bit，byte(8bit, -128-127)、short(16bit)、int(32bit)、long(64bit)、单精度(32bit,float)、双精度(64bit,double)、boolean(8bit)、char（Unicode，16bit）

    3. 例如：int i = 8143转换成字节数组。8143的二进制位 00000000 00000000 00011111 1100111。 取得第一个低端字节：8143 >> 08 & 0xff(11111111,前边补0) = 1100111。把8143右移 08 位，然后按位与0xff，也就是（00000000 00000000 00000000 11111111）取得最后第一个低端位的二进制数 1100111，转换为十进制就是207或者有符号-49。第二个低端字节：8143 >> 1*8 & 0xff。8143右移8位之后舍弃左边8位，右边补0，然后按位与0xff就得到了00011111。第三、第四一样。

    4. 小端法（little-endian）与大端法（big-endian）

      • 小端法既是将低位字节排放在内存的低地址段即该值得起始端。

      • 大端法既是将高位字节排放在内存的低地址段即该值的起始端。

    5. 字符串与字节数组的转换

      • 字符串转字节数组：只需要通过String类的getBytes方法实现。

      • 字节数组转字符串：只需通过String类的构造方法即可，即new一个String类把字节数组当做参数即可。还可以制定编码格式。

    当一个类实现了序列化接口，它的子类都可以实现序列化，当一个子类进行序列化时，会递归调用父类的构造方法，当一个子类被反序列化时，会递归调用父类没有实现序列化接口的构造方法。原因是如果父类实现了序列化接口，在序列化时会把父类的构造方法也序列化，反之则会在反序列化时调用父类的构造方法。
